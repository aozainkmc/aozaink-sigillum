# aozaink-sigillum 首版玩法设计稿（计划）

> 本稿只描述"做什么"与"怎么玩"，不描述实现代码。所有机制应围绕「手写汉字 → 黄符成式 → 施法/刻印」这条主轴展开。普通黄符最多支持两个技能字，并用第三格尾修承载修饰变化。
> **文档分工**：本文件是玩法计划和后续联动设计；当前代码事实记录在 `IMPLEMENTATION.md`。没有特别标注“当前实现”的联动描述，都不应视为已经进代码。

---

## 一、首版目标

让玩家能用自己手写的汉字，在黄符上组合出不同术式，然后：

1. **主手使用黄符** → 对实体/区域释放一次性术式；
2. **把黄符刻印到方块上** → 形成持续一段时间的场地效果；
3. **用快速吟唱字** → 把常用字绑定到中文数字字 一～九，加快施法节奏。

所有术式都围绕"英叔风格"的驱邪/镇煞/护身/引灵展开，不做法杖、不做召唤、不做大型仪式。

## 二、字表：首版 16 字（12 技能 + 4 修饰）

黄符只有三格，每格写一个字。首版严格提供 **16 个玩法字**（12 技能字 + 4 修饰字）：

| 类别 | 字 | 单字效果 | 组合中的作用 |
|---|---|---|---|

| **封禁** | 封 | 封行动/沉默，阻止靠近 | 基础技能，可参与联动 |
| **驱退** | 退 | 击退目标、驱邪退散 | 基础技能，可参与联动 |
| **牵引** | 引 | 拉扯目标向施法者 | 基础技能，可参与联动 |
| **元素** | 火 | 造成持续火焰伤害，点燃目标 | 基础技能，可参与联动 |
| **元素** | 雷 | 真实落雷触发原版雷击判定，附带爆发伤害和短硬直 | 基础技能，可参与联动 |
| **守护** | 护 | 自身吸收护盾 | 基础技能，可参与联动 |
| **净化** | 净 | 清除负面状态，对亡灵造成圣化伤害 | 基础技能，可参与联动 |
| **斩杀** | 斩 | 目标血量低于阈值时以玩家伤害处决 | 基础技能，可参与联动 |
| **照妖** | 明 | 自身夜视；瞄准生物时单体发光 | 基础技能，可参与联动 |

| **招魄** | 魄 | 召回上次死亡掉落物 | 基础技能，可参与联动 |

| **强化** | 续 | 全局延长符上所有技能字的持续时间（1 + 品质倍率） | 全局修饰 |

**功能字（不计入 12 个玩法字）：**

| 字 | 作用 |
|---|---|
| **刻** | 写在前两个咒位之一，表示该符用于刻印到方块 |
| **一～九** | 用于快速吟唱绑定与触发，本身不产生效果 |

## 三、黄符格位语义

黄符 UI 的三格不是三个等价技能位，而是：

```
[咒位] + [咒位] + [尾修]
```

- **咒位 1/2**：可写技能字或修饰字；
- **尾修**：只允许写修饰字（强/续/广/穿）或留空；不接受技能字、刻、数字字；
- 普通组合符最多只有 **2 个技能字**，不存在“3 技能字组合”；
- 修饰字全局作用于符上所有技能字；
- 多个修饰字可叠加，但广/穿互斥，同时出现判废符；
- 空格不写任何字，表示“跳过”。例如「镇 + 空 + 空」就是最简单的单体镇压。

普通组合符的合法结构只有以下几类：

| 结构 | 示例 | 含义 |
|---|---|---|
| 技能 | 火 | 单字施法 |
| 技能 + 修饰 | 火 + 强 | 单技能受一个修饰 |
| 技能 + 修饰 + 修饰 | 火 + 强 + 续 | 单技能受两个修饰 |
| 技能 + 技能 | 火 + 雷 | 二字特殊联动 |
| 技能 + 技能 + 尾修 | 火 + 雷 + 强 | 二字组合带一个尾修 |

不合法结构：

- 第三格写技能字（UI 应直接禁止，已有旧物品也不作为设计目标）；
- 三个技能字；
- 没有技能字（纯修饰字 / 纯数字字）；
- 同一玩法字重复出现；
- 广和穿同时出现；
- 穿作用于两个技能字（当前阶段判废，除非后续为某个二字组合单独设计）。

### 刻印符的特殊格式

若黄符前两个咒位中包含且只包含一个 **「刻」**，则该符被视为**刻印符**。刻印符沿用普通黄符的格位约束：前两个是咒位，第三格仍是尾修槽。

```
[刻/技能字/修饰字] + [刻/技能字/修饰字] + [修饰字/空]
```

- 「刻」本身不占玩法效果位，只表示"这张符用于刻印到方块"；
- 「刻」只能在前两个咒位，不能写在尾修槽；
- 刻印符天然以方块为锚点，不需要再写形态字；
- 除「刻」外，最多允许一个技能字；若没有技能字，则最多允许两个合法修饰字用于操作已有刻印；
- `刻 + 技能字 + 修饰字` 与 `技能字 + 刻 + 修饰字` 等价；
- `刻 + 技能字` 创建默认刻印；
- `刻 + 修饰字` 只用于操作已有刻印；
- `刻 + 修饰字 + 修饰字` 可对已有刻印连续执行两个合法操作；
- `刻 + 技能字 + 技能字` 无法由规范 UI 产生，旧物品也按废符处理。

## 四、基础单字效果

本节列出每个技能字的**单字基础效果**。组合符可以按后文联动表把两个基础效果合成为新术式；修饰字单独成符为废符：没有技能字可修饰，无法形成有效黄符。

### 镇 · 封 · 退 · 引 · 斩 · 明 · 吸 · 魄

- **镇**：目标 3 秒减速 II + 攻击降低 I；亡灵增强 50%。镇压/定身/镇邪。
- **封**：目标 3 秒无法普攻（沉默）或区域被封闭无法通过；亡灵额外 1.5 秒硬直。封门/封路/封印。
- **退**：目标被击退 3 格 + 1 秒僵直；亡灵击退距离翻倍。驱退/驱邪。
- **引**：目标被拉起向施法者，拉力随距离递减；玩家无效。
- **斩**：目标当前血量 ≤ 50%(极品)/25%(良品)/10%(劣质) 时以玩家伤害处决，走正常死亡、掉落、经验与击杀归属；未触发则产生烟雾提示并消耗。百分比斩杀不怕高血量怪物。
- **明**：自身获得长夜视；若视线命中生物，则给该生物施加发光。未命中也可生效为夜视。`穿` 让视线上的多个生物发光；`广` 才改为范围照妖，但缩短夜视/发光时长。用于照妖、反隐身、夜间作战。
- **吸**：命中目标造成魔法伤害并回血；极品回 1.5 心(3 点)，良品 1 心(2 点)，劣质 0.5 心(1 点)。
- **魄**：召回上次死亡时掉落的物品。死亡时记录死亡坐标与维度；右键后把死亡点半径 48 格内归属自己的掉落物传送回身边。极品 100% 回收，良品 75%，劣质 50%。

### 火 · 雷

- **火**：目标点燃 4 秒，满效果 **2 点/秒**（亡灵 **3 点/秒**），随品质缩放；v1 为视线单体命中。
- **雷**：生成真实闪电实体，先触发原版雷击判定（苦力怕充能、猪/村民等原版转化），随后补单体雷击满效果 **12 点**；雨天室外 ×1.3，50% 附带 0.5 秒硬直；额外伤害无视目标无敌帧、全额结算。随品质缩放。天雷定位为高于火术的爆发单体。

### 护 · 净

- **护**：自身获得独立额外护盾条；极品 **30 点**、良品 **16 点**、劣质 **8 点**，不会随时间消失，只在受击时扣除，扣完才开始扣原版血条。刻印方块时形成「金光符界」防护罩。
- **净**：清除自身负面（**极品 3 / 良品 2 / 劣质 1** 个）；瞄准亡灵额外满效果 **4 点**圣化伤害（随品质缩放）。品质对净的体现 = 清除条数 + 圣化伤害。

## 四·五 品质评级（整张符一个品级）

每张成符黄符最终只产生**一个整符品级**。识别/评分阶段可以保留每格分数用于 UI 诊断，但玩法结算时不存在“某个字极品、另一个字劣质”的局部效果。若整张符是极品，则符上所有技能字和修饰字都按极品效果执行；若整张符是良品/劣质，也全部按同一个品级执行。

| 品质 | 效果倍率 | 何时达到（input 侧阈值） |
|---|---|---|
| 极品 | 100%（满效果） | 综合分 ≥ 0.95 |
| 良品 | 80% | 综合分 ≥ 0.75 |
| 劣质 | 55% | 综合分 ≥ 0.50 |
| 废符 | 0%（不施放、消耗符） | 综合分 < 0.50 |

上方列出的所有效果数值均为**满效果（极品）**，低品质按倍率线性缩放。

- **整符品级生成**：每个非空格仍可计算 `slotComposite` 用于调试和提示；最终 `talismanComposite` 暂定取所有非空格 `slotComposite` 的最小值，再映射为一个 `talismanGrade`。这个最小值只决定整张符品级，不会让某个字单独变弱。
- **整符品级结算**：sigillum 只读取一个 `talismanGrade/finalM`。技能字、修饰字、联动效果全部使用同一个 `finalM`。禁止在玩法结算里读取“第 1 格品质/第 2 格品质”来分别缩放某个字。
- **修饰字受整符品级影响**：强/续/广/穿 的收益也随整符品级缩放（强=`finalM × 2.0`，续=`1 + finalM`，等），品质差时修饰收益同步降低。广与穿互斥，同时出现判废符。护盾没有时间衰减，因此续对护无收益。
- **评分公式（input 侧的 TalismanScorer）**：`composite = confidence³ + (笔画扣分) + (点数扣分) + (时长扣分)`，与标准范本比对得出 0~1.3 范围分，再映射到四档。
- **缩放方式（计划目标）**：sigillum 把整符品级换成 `finalM`，`续` 额外给 `durationMultiplier`。具体技能只有读取对应通道才会变强或变久，完整当前实现事实见 `IMPLEMENTATION.md`。
- **品质写在黄符物品上**：tooltip 直接显示整符「品质: 极品/良品/劣质/废符」；调试模式可以额外显示每格诊断分，但不作为玩法品级。**同组合 + 同类型 + 同整符品质的符可叠加**（每组 64）。
- **结构废符（乱写没成式）与品质废符（成式但分太低）统一**：右键都消耗并 action bar 提示「废符」。
- **实现分工**：input 负责客观评分并写入整符品级字段；sigillum 读整符品级换成效果倍率施放。若为了兼容旧物品暂时保留 `CustomData.aozaink:grade<格>`，这些字段也应镜像同一个整符品级，不能表达每字不同品级。

### 修饰字规则

修饰字本身不生效，全局作用于黄符上**所有技能字**：

| 字 | 修饰效果（当前实现摘要） |
|---|---|
| 强 | 把效果倍率设为 `overallM × 2.0`；只影响读取 `multiplier` 的技能 |
| 续 | 把持续倍率设为 `1 + overallM`（极品 ×2.0 / 良品 ×1.8 / 劣质 ×1.55）；只影响读取 `durationMultiplier` 的技能 |
| 广 | 需要至少一个目标效果。普通目标效果从单体变范围（半径 4 格），范围效果倍率再 ×0.7；混合符中的自施效果仍按自身逻辑结算；明例外为施法者中心范围照妖 |
| 穿 | 只支持单个目标效果的穿透射线，极品 3 个/良品 2 个/劣质 1 个目标；命中后降档复用，未命中原样掉回地上；多技能组合时禁用穿；与广同时出现判废符 |

多个不同修饰字通常可以同时生效并叠加，但 **广/穿互斥**，同时出现直接判废符。**所有修饰字收益都受整体品质倍率缩放**（品质差则修饰收益同步降低，防止"品质烂但带强就超模"）。修饰字单独成符为废符。同一玩法字重复出现不表达强化，判为废符；强化应使用 `强/续/广/穿`。

### 双技能特殊联动设计表

当前代码已为普通组合符实现 12 选 2 = 66 种双技能联动，顺序无关；下表保留设计语义。实际代码细节、近似实现和当前边界以 `IMPLEMENTATION.md` 为准。

表格写给开发者看，只描述目标、触发顺序、额外判定和资源归属：
- “主目标”指本次符命中的 LivingEntity。
- “施法者”指使用黄符的玩家。
- 所有数值、阈值、持续时间和修饰收益都读取**整张符的最终品级**，不存在按单个字分别取品质。
- `魄` 始终只召回施法者上次死亡掉落，不吸走本次被攻击目标的掉落。
- 写明“按实际伤害治疗”时，治疗量基于该次事件最终造成的有效伤害。
- 标注 `直接组合` 的行不需要额外联动分支；实现时按两个单字效果同时结算，但仍使用整符品级、同一次消耗和同一次命中上下文。

| 组合 | 计划结果 |
|---|---|
| 镇 + 封 | 合成 `镇封` 状态：主目标移动速度被压到近似定身，同时封禁攻击/挖掘/特殊动作；持续按整符品级计算，不再分别套两个普通控制 |
| 镇 + 退 | 先执行退；目标落地或碰撞后立刻施加短镇压，防止被击退后马上重新贴近 |
| 镇 + 引 | 牵引不再拉到玩家脚下，而是把主目标拉到玩家前方安全距离后镇压，形成可控拉怪点 |
| 镇 + 火 | 主目标获得镇压燃烧：火 DoT 持续期间若镇压仍在，燃烧不会提前结束；火伤仍按整符品级计算 |
| 镇 + 雷 | 主目标成为雷锚：雷击后硬直必定触发；若附近有敌对怪，向最近 1 个目标跳一次低倍率雷链 |

### 单技能 + 双修饰

单技能双修饰是受控增长面，不需要为每个组合写独立技能设计。共有 `12 × C(4,2) = 72` 种结构，按修饰字框架叠加：

| 修饰组合 | 通用规则 |
|---|---|
| 强 + 续 | 效果倍率提高，并延长可持续部分；无持续效果的技能只吃强 |
| 强 + 广 | 范围化后提高效果倍率；广仍使用范围衰减 |
| 强 + 穿 | 穿透命中多个目标，每个目标按强化后的效果结算；命中后仍按穿规则降档 |
| 续 + 广 | 范围化并延长可持续部分 |
| 续 + 穿 | 穿透命中多个目标，并延长可持续部分 |
| 广 + 穿 | 互斥，直接判废符 |

纯自施技能若没有目标效果，仍不能吃广/穿。当前例外是 **明**：它同时具备自施夜视和目标显形，所以明可使用广/穿。

### 二技能 + 尾修

二技能 + 尾修 是后续主要扩展面，共 `66 × 4 = 264` 个格点。不能只写“强/续/广/穿通用套用”，因为二字联动里既有直接组合，也有转目标、召回、护盾事件、范围标记等特殊语义。实现时必须按下表逐 pair 取策略。

通用实现约束：

- `强` 可以通过 `CastEnv.multiplier` 进入二字联动，但每个 pair 必须定义“放大什么”；不能让联动内部两个技能各自重复吃强导致双重翻倍。
- `续` 可以通过 `CastEnv.durationMultiplier` 进入二字联动，但每个 pair 必须定义“延长什么”；纯瞬时/召回部分可无收益。
- `广` 只改变作用形态：若原二字联动仍是非自施单体效果，则以主目标为中心范围化；若原二字联动已经是范围/路径/扫描效果，则只扩大原半径、宽度或搜索范围。`广` 不新增额外玩法。
- 二技能尾修中若联动含 `护`/`净` 这类自施支援资源，`广` 可把支援溢出给范围内队友：先补齐施法者自己的效果；施法者吃不下的护盾量/净化次数按距离从近到远给队友；队友满了继续给下一个；全员都吃不下则丢弃。支援总量按本张黄符品质生成，沿用原有数值，不因队友人数复制。
- `穿` 不做全局通用。只有表中明确写出路径语义的 pair 才可实现；其余判废符。
- 当前代码事实以 `IMPLEMENTATION.md` 为准；本表是计划/实现目标，不代表全部已落地。

| 二字组合 | +强 | +续 | +广 | +穿 |
|---|---|---|---|---|

| 明 + 魄 | 提高明标记时长；魄回收比例不重复 | 延长明标记，魄无持续收益 | 已是明范围标记语义；广只扩大明范围，魄一次 | 废符：魄不支持穿透 |
| 吸 + 魄 | 提高吸伤和治疗；魄回收比例不重复 | 吸无持续收益，魄无持续收益 | 吸目标效果范围化；魄只执行一次 | 废符：魄不支持穿透 |

### 不存在的结构：三技能字组合

普通符不设计三技能字组合。原因：

- UI 第三格是尾修槽，代码层应禁止技能字进入；
- 三技能组合会引入 `C(12,3)=220` 个额外结构，且多数只是三个单字并排执行，设计感低；
- 二技能联动已经提供 66 个核心组合，尾修再提供强度/时长/范围/穿透四类变化，玩法容量已经足够；
- 后续如果需要更复杂效果，应设计为“二技能联动 + 尾修覆盖”，而不是恢复三技能字。

### 刻印符全量组合

刻印符前两个咒位中包含且只包含一个「刻」；第三格仍是尾修槽，只能为空或写修饰字。除「刻」外最多一个技能字；若没有技能字，则最多两个合法修饰字可作为已有刻印操作。刻印以目标方块为锚点保存到服务端维度数据，随世界保存/读取；服务器暂停、退出游戏、区块未加载时不扣 tick。

#### 刻印基础资源

- 默认持续时间/能量为 **36000 tick（30 分钟）**，按整体品质倍率缩放。
- 刻印每过 1 个游戏 tick 扣 1 点剩余时间；每对一个目标结算一次技能，额外扣 1 点能量。
- 时间或能量任意一项归零，刻印消失。
- 单个刻印的剩余时间封顶 **24 小时 = 1,728,000 tick**；普通刻印能量同上限。
- `刻 + 护` 是特殊防护罩，初始能量为普通刻印的 **5 倍**，能量上限也按 5 倍计算。
- 同一方块只能有一个刻印；不同方块的范围重叠时各自独立结算，同一只怪可被多个刻印分别命中。

#### 刻印叠加

- `刻 + 续` 可对准已有刻印方块续时/补能；剩余时间 +36000 tick（按符品质倍率），能量按该刻印能量系数补充，均受上限限制。
- `刻 + 续` 若没有对准已有刻印方块，不消耗并提示需要对准带刻印的方块。
- `刻 + 强` 可对准已有刻印方块加强一次；未加强过时当前剩余时间和能量翻倍（受上限限制），并标记为已加强。已加强过则不再叠加。
- 首次创建时带 `强` 也视为已加强，基础时间/能量直接翻倍。
- `刻 + 广` 创建时把刻印半径直接翻倍；后续对已有刻印使用 `刻 + 广` 会把范围提高到该符品质对应的翻倍半径。
- `刻 + 穿` 永远判废符；刻印不支持穿透。

#### 刻 + 技能字

默认半径按品质决定：极品 10 格、良品 8 格、劣质 5 格；`广` 后分别为 20/16/10 格。默认持续 30 分钟。每个进入范围并满足冷却的实体由该刻印单独结算一次，结算后记录实体 UUID，避免同一 tick 反复触发；实体离开范围或冷却到期后可再次触发。刻印存续期间通过客户端世界 UI 显示金色剩余条；普通刻印只显示贴地深朱砂边界，目标类刻印对范围内敌对目标显示深朱砂锁定线，自身/工具类刻印不显示锁定线；`刻 + 护` 独占三维球形金光符界。

| 组合 | 基础效果 |
|---|---|
| 刻 + 镇 | 范围内目标持续续杯减速/虚弱 |
| 刻 + 封 | 范围内目标短封禁；每约 2 秒重新锁定，允许目标缓慢接近 |
| 刻 + 退 | 每约 10 tick 检测范围内目标并执行一次向外击退脉冲，无伤害 |
| 刻 + 引 | 范围内目标持续受到向中心场地力，缓慢加速靠近刻印核心；周期结算时追加一次较强牵引 |
| 刻 + 火 | 周期性点燃范围内目标 |
| 刻 + 雷 | 周期性真实落雷并补雷击伤害 |
| 刻 + 护 | **收口确认：金光符界防护罩。** 玩家在范围内持续获得独立护盾；非玩家投射物撞界泯灭并扣盾；敌对生物在边界被挡住但不反弹，界内入侵物静滞消磨；消磨伤害为魔法体系，按间隔扣盾。护罩视觉为静态球形金色节点/连线/薄幕，亮度随剩余时间与能量进度变化 |
| 刻 + 净 | 周期性净化范围内玩家负面效果，并圣化伤害亡灵 |
| 刻 + 斩 | 对范围内低血量目标周期性执行斩杀判定 |
| 刻 + 明 | 范围内玩家获得夜视，敌对目标发光 |
| 刻 + 吸 | 周期性伤害目标并为刻印主人回血 |
| 刻 + 魄 | 主人在范围内时尝试召回死亡掉落 |

#### 刻 + 技能字 + 修饰字

修饰字全局修饰该单技能刻印。`强` 创建时视为已加强，`续` 创建时提高初始时间/能量并可用于后续续时，`广` 扩大范围，`穿` 判废符。`刻 + 技能字 + 修饰字` 与 `技能字 + 刻 + 修饰字` 等价。第三格不能写技能字，因此不存在 `刻 + 技能字 + 技能字`。

#### 刻 + 修饰字

只允许用于已有刻印的操作：`刻 + 续` 续时/补能，`刻 + 强` 加强一次，`刻 + 广` 扩大一次范围，`刻 + 穿` 判废符。若目标方块没有刻印，提示需要对准带刻印的方块。

#### 刻 + 修饰字 + 修饰字

只允许用于已有刻印的连续操作。合法组合为 `强 + 续`、`强 + 广`、`续 + 广`；按书写槽位顺序依次执行。若任一修饰字为 `穿`，整张符判废，因为 `刻 + 穿` 单独就是废符。

#### 刻 + 数字字

数字字不是技能字、不是修饰字，刻印符含数字字 → 废符。

### 指定符

指定符只用于快速吟唱绑定，不产生战斗效果：

| 组合 | 结果 |
|---|---|
| 数字 + 技能字 + 空 | 将技能字绑定到该数字 |
| 数字 + 空 + 空 | 解除该数字的绑定 |

### 废符

以下情况无法形成有效黄符，右键产生轻微火花：
- 三格全空
- 普通符没有任何技能字（纯修饰字 / 纯数字字）
- 同一玩法字在一张符中重复出现（重复技能字或重复修饰字）
- 刻印符含数字字
- 第一格为数字字但格式不匹配指定符（如数字 + 数字 + 空）
- 刻印符只有「刻」且没有修饰操作
- `刻 + 穿` 或任何带「穿」的刻印符
- 「刻」写在尾修槽
- `刻 + 技能字 + 技能字`

## 六、评分与黄符品级（计划：整符单品级）

手写质量决定黄符品级。**评分在 input 侧**（`TalismanScorer`）完成，sigillum 只读取最终的**整符品级**。

### 评分链路

```
玩家手写 → core 输出(置信度) + input 采集(笔画数/点数/时长)
    → TalismanScorer.scoreSlot() 计算每格诊断分
    → 聚合为 talismanComposite
    → 映射为整符品质等级 → 写入整符 grade 字段
    → sigillum 读取整符 grade → TalismanGrade 换算效果倍率
```

### 评分公式

`TalismanScorer` 可以对每格独立计算诊断分，但玩法只使用聚合后的整符分：

```text
slotComposite = confidence³ + strokePenalty + pointPenalty + timePenalty
talismanComposite = min(nonEmptySlotComposite...)    # 暂定聚合方式
```

- **置信度立方**：模型置信度做立方运算——高置信度（≥0.9）接近满分，低置信度（≤0.6）急剧下降，惩罚"模型拿不准"的字。
- **笔画扣分**：实际简化笔画数与标准笔画数的偏差，超出 ±3 画扣 10%。
- **点数扣分**：简化点数与标准的偏差，按比例扣减（最多 -3%）。
- **时长扣分**：书写时长与标准基准的偏差，过快过慢都扣分（最快 -10%）。
- **综合分范围**：0 ~ 1.3（快写可略超满分 1.0，但封顶不影响定级）。

### 品质阈值与倍率

| 品质 | 综合分区间 | 效果倍率 | display |
|---|---|---|---|
| **极品 (EXQUISITE)** | ≥ 0.95 | ×1.0（满效果） | "极品" |
| **良品 (FINE)** | ≥ 0.75 | ×0.8 | "良品" |
| **劣质 (INFERIOR)** | ≥ 0.50 | ×0.55 | "劣质" |
| **废符 (WASTE)** | < 0.50 | ×0.0（不放，消耗） | "废符" |

### 整符品质

每张符最终只有一个品级，所有技能字和修饰字都读取这一个品级：
- 三格诊断分聚合后为极品 → 符上每个字都按极品执行
- 三格诊断分聚合后为良品 → 符上每个字都按良品执行
- 三格诊断分聚合后为劣质 → 符上每个字都按劣质执行
- 如果某格写得差，只会拉低整张符的最终品级，不会只削弱该格对应的字

品质阈值与倍率是 sigillum 侧常量（`TalismanGrade` 枚举），input 侧也存一份一致的阈值用于成符时定级。**改名要两边同步**。如果旧物品格式仍有 `grade1/grade2/grade3`，它们只能作为兼容镜像或诊断字段，不能驱动单字强弱。

### 标准范本

校准基线存储在 `assets/aozaink_input/scoring/reference.json`，由 `tools/ink_calibrator/` 工具生成。每个字记录标准笔画数、点数、参考时长（从用户手写样本采集）。

### 品质显示

- 黄符 tooltip 显示整符「品质: 极品/良品/劣质/废符」——无需使用、无需 dev 即可见。
- **同组合 + 同类型 + 同整符品质的符可叠加**（每组 64）。
- 结构废符（乱写未成式）与品质废符（成式但分低）统一：右键消耗 + action bar 提示「废符」。

## 七、黄符两种使用方式

> **实现状态**：①直接施法 全部 12 技能字 + 4 修饰字 已实现。②刻印符已实现服务端持久范围效果、续时/补能、一次加强、广域扩展与穿透废符规则；刻印符当前限定为单技能刻印或单修饰操作，不做双技能刻印。

### 1. 直接施法

主手持已成符黄符，右键释放：

- 对实体释放时，术式直接作用于该目标（单体）；
- 对空气/地面释放时，术式以玩家前方为中心生效；
- 作用范围由符上技能字效果 + 修饰字共同决定：
  - 默认单体；
  - 含「广」时扩展为范围或影响多个目标；
  - 含「穿」时变为穿透射线；
- 对方块释放时，若黄符是刻印符则进入刻印流程，否则按"对地面释放"处理。

### 2. 刻印到方块

持刻印符右键方块，把术式刻印在方块上：

- 一个方块**只能同时存在一个刻印**；
- 若目标方块已有刻印，`刻 + 续` 续时/补能，`刻 + 强` 最多加强一次，`刻 + 广` 扩大范围；其他新刻印拒绝覆盖；
- 刻印后黄符被消耗；
- 方块被破坏时刻印失效；区块未加载时刻印暂停，不扣时间。

刻印符前两个咒位中包含且只包含一个「刻」，第三格仍是尾修槽。除「刻」外按规范格式解析：
- **一个技能字**：产生对应技能的周期性脉冲效果；
- **一个技能字 + 一个修饰字**：修饰字全局修饰该刻印的技能效果，`穿` 除外；
- **只有修饰字**：可对已有刻印执行 `续/强/广` 操作，最多两个合法修饰字；按槽位顺序连续执行；
- **两个技能字 / 刻在尾修槽 / 数字字 / 重复玩法字 / 带穿刻印**：废符。

## 八、快速吟唱字

玩家可以把一个常用字绑定到中文数字字 一～九，例如：

- `/sigillum dev` 开启开发模式后，用 `/sigillum bind 1 雷` 把"雷"绑定到"一";
- 战斗中写一个"一"字，自动视为"雷"，缩短前摇。

快速吟唱规则：

- 写绑定字时，效果等同写原字;
- 前摇缩短 30%，但法耗或冷却略微增加（若后续加入资源系统）;
- 若当前未绑定，则按字面意思识别。

## 九、原版成就系统

成就使用 Minecraft 原版 Advancement 系统，不做熟练度、不做独立奖励树、不引入数值成长。成就树基于当前已经完成的玩法面设计：黄符放置/书写、成符类型、品质、拓印、指定符和快速吟唱、12 个技能字、66 对二字联动、二字 + 尾修、单技能 + 双修饰、尾修暴乱、护盾金色条、魄召回、持久刻印和刻印操作。

成就不逐个覆盖 66 对组合或 264 个二技能 + 尾修格点。完整玩法由黄符预览、tooltip 和设计表承载；成就只标记玩家进入某个玩法分支、完成一种高辨识度结果，或触发少数隐藏彩蛋。

### 成就树结构设计

主树放在 `aozaink_sigillum` 命名空间。所有自定义 criterion 由 sigillum 注册并触发；input 只在服务端广播客观输入结果信号，sigillum 监听后解释为成就。core 不放成就语义，只继续作为识别、服务注册和通用信号事件底座。

成就树不做一条长主线。结构参考原版“农牧业”页：根节点在左，右侧分出多个横向分支；每个分支的父节点必须是该玩法自然先触发的宽条件，不能用“合成空白黄符”之类无关节点去卡住施法、刻印、尾修等分支。玩家先体验哪个玩法分支，就应该能立刻拿到该分支的入门成就。

### 自定义触发器规划

Advancement JSON 只描述显示、父子关系、图标、标题和描述；玩法模块在服务端关键点触发 criterion。触发器要服务端权威：只在实际物品产出、实际消耗、实际命中、实际刻印状态变化或实际伤害吸收之后触发，不因 UI 预览或客户端粒子触发。

| trigger id | 触发模块 | 参数 | 覆盖成就 |
|---|---|---|---|
| `minecraft:recipe_crafted` | input | `recipe_id=aozaink_input:yellow_talisman / aozaink_input:talisman_copy` | 空白黄符合成、拓印 |
| `aozaink_sigillum:talisman_created` | input signal -> sigillum | `type`、`grade`、是否带尾修 | input 广播客观成符结果，sigillum 解释为成就 |
| `aozaink_sigillum:tail_modifier_chaos` | input signal -> sigillum | 是否炸到玩家、是否炸死实体、玩家是否逃出 | input 广播尾修暴乱结果，sigillum 解释为成就 |
| `aozaink_sigillum:specified_bound` | sigillum | 技能字 | 指定符绑定 |
| `aozaink_sigillum:quick_cast` | sigillum | 绑定字 | 快速吟唱 |
| `aozaink_sigillum:talisman_cast` | sigillum | 类型、技能字、修饰字、命中数、是否 linked combo、特殊枚举 | 直接施法、单技能、二字、尾修、广/穿 |
| `aozaink_sigillum:inscription_changed` | sigillum | `created/extended/strengthened/widened/depleted/full_day`、技能字、修饰字 | 刻印创建、续时、强化、扩大、耗尽、24 小时 |
| `aozaink_sigillum:soul_recalled` | sigillum | 物品数 + 经验量 | 魄召回 |
| `aozaink_sigillum:shield_event` | sigillum | `absorbed/shared`、数值 | 护盾吸收、队友支援 |
| `aozaink_sigillum:special_effect` | sigillum | `lightning_transmutation/slash_mark_miss_all/fire_thunder_chain` 等枚举 | 少数彩蛋效果 |

### 成就显示原则

- 可见节点标题短，描述直接告诉玩家对应玩法结果，不写大段说明。
- 隐藏节点可以有更有味道的标题，但触发条件必须明确写在设计文档里，避免后续实现猜测。
- 背景使用黄符纸色系，素材来源应来自黄符 UI 的咒位视觉语言，但必须注意 advancement 背景会平铺；不能直接把完整咒位图案当 16x16/64x64 背景重复铺满。推荐方案是纸色底 + 极低对比度的咒位线稿纹理，线条 alpha 约 4%-8%，整体不能抢过节点图标。
- 不给数值奖励；如要给经验或物品，必须另开设计讨论，不能混进成就系统。
- 不把 66 个二字组合逐个做成就；只挑高辨识度联动作为隐藏彩蛋。
- 不把“累计释放 1000 次”这类纯刷次数作为当前成就目标。若后续需要统计型挑战，放到扩展章节。

## 十二、体验节奏

一次完整的符术循环应该是：

1. 玩家准备空白黄符（3 纸 + 1 黄色染料合成 6 个）；
2. 把黄符放在工作台上，右键打开三格书写 UI；
3. 用鼠标在三格里手写字，每格停止书写 0.7 秒后自动识别；
4. 若要做刻印符，前两个咒位之一写「刻」，尾修槽只能留空或写修饰字；
5. 点击"成符"，黄符被写入三格字与品级分；
6. 主手使用黄符施法，或持刻印符右键方块刻印；
7. 观察效果，下次调整写法以提高品级。

失败也应有反馈：

- 识别失败：黄符变成"废符"，右键只产生轻微火花；
- 品级太低：效果打折，玩家能从 HUD 看到自己写的是"下品"；
- 多修饰字：普通符上多个不同修饰字通常全局叠加生效（受三格限制最多 2 个），但广/穿互斥，同时出现判废符；同一修饰字重复写判废符。刻印符创建单技能刻印时只容纳一个修饰字；对已有刻印操作时可写两个合法修饰字，含穿则废符。

## 十四、设计原则

- **服务端权威**：所有效果结算在服务端执行，客户端只负责输入与表现；
- 识别结果可来自客户端（core 在客户端完成 OCR），但效果结算与刻印状态必须服务端权威；
- **数据驱动**：字表与联结表尽量用配置文件或数据包描述，减少硬编码；
- **组合可预测**：二字联动按 exact pair 查表，尾修作为外层策略进入，不靠运行时抢解释权；
- **反馈清晰**：玩家能知道自己写的是什么品级、会触发什么术式、为什么失败；
- **扩展友好**：新增字或联结不应改动现有编译器骨架。
