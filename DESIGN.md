# aozaink-sigillum 首版玩法设计稿

> 本稿只描述"做什么"与"怎么玩"，不描述实现代码。所有机制应围绕「手写汉字 → 黄符成式 → 施法/刻印」这条主轴展开，最多支持 3 字联动。

---

## 一、首版目标

让玩家能用自己手写的汉字，在黄符上组合出不同术式，然后：

1. **主手使用黄符** → 对实体/区域释放一次性术式；
2. **把黄符刻印到方块上** → 形成持续一段时间的场地效果；
3. **用快速吟唱字** → 把常用字绑定到中文数字字 一～九，加快施法节奏。

所有术式都围绕"英叔风格"的驱邪/镇煞/护身/引灵展开，不做法杖、不做召唤、不做大型仪式。

黄符只有三格，每格写一个字。首版严格提供 **12 个玩法字**，分为两类：

|---|---|---|---|

| **驱退** | 退 | 击退目标、驱邪退散 | 独立效果 |
| **牵引** | 引 | 拉扯目标向施法者或符阵中心 | 独立效果 |
| **元素** | 火 | 造成持续火焰伤害，点燃目标 | 独立效果 |
| **元素** | 雷 | 爆发雷电伤害，附带短硬直 | 独立效果 |

## 三、三格语义

黄符的三格是**视觉位置（左中右）**，不代表技能顺序。所有技能字**无序平等**，修饰字**全局生效**于所有技能字：

- 所有技能字各生效各的，同时触发；
- 修饰字同时作用于黄符上的**所有技能字**；
- 多个修饰字同时生效，效果叠加；
- 空格不写任何字，表示"跳过"。例如「镇 + 空 + 空」就是最简单的单体镇压。

### 刻印符的特殊格式

若黄符第一格为 **「刻」**，则该符被视为**刻印符**。「刻」后两格按无序规则解析：

```
刻 + [技能字/修饰字] + [技能字/修饰字/空]
```

- 「刻」本身不占玩法效果位，只表示"这张符用于刻印到方块"；
- 刻印符天然以方块为锚点，不需要再写形态字；
- 后两格必须至少包含一个技能字；
- `刻 + 技能字 + 修饰字` 与 `刻 + 修饰字 + 技能字` 等价；
- 两个技能字表示两个独立刻印脉冲；
- 两个修饰字没有技能字可供刻印，判为废符；
- 若后两格只有一个技能字，使用默认参数。

## 四、基础单字效果

每格独立贡献效果。修饰字单独成符为废符：没有技能字可修饰，无法形成有效黄符。

### 镇 · 封 · 退 · 引

- **镇**：目标 3 秒减速 II + 攻击降低 I；亡灵增强 50%。镇压/定身/镇邪。
- **封**：目标 3 秒无法普攻（沉默）或区域被封闭无法通过；亡灵额外 1.5 秒硬直。封门/封路/封印。
- **退**：目标被击退 3 格 + 1 秒僵直；亡灵击退距离翻倍。驱退/驱邪。
- **引**：目标被拉起向施法者，拉力随距离递减；玩家无效。

### 火 · 雷

- **火**：目标点燃 4 秒，满效果 **2 点/秒**（亡灵 **3 点/秒**），随品质缩放；v1 为视线单体命中。
- **雷**：单体雷击满效果 **12 点**，雨天室外 ×1.3，50% 附带 0.5 秒硬直；**每发无视目标无敌帧、全额结算**（连发不被吃伤害）。随品质缩放。天雷定位为高于火术的爆发单体。

### 护 · 净

- **护**：自身满效果 **10 点**吸收护盾，10 秒，随品质缩放。（刻印方块防护罩为后续）
- **净**：清除自身负面（**极品 3 / 良品 2 / 劣质 1** 个）；瞄准亡灵额外满效果 **4 点**圣化伤害（随品质缩放）。品质对净的体现 = 清除条数 + 圣化伤害。

## 四·五 品质评级（写得好＝效果强；v1 已接入火/雷/护/净）

每次书写黄符，系统按"写得有多好"给**每个字**一个**品质等级**，决定该字术式的**效果倍率**。上方各效果列出的都是**满效果（极品）**数值，低品质按倍率缩放。

| 品质 | 效果倍率 | 何时达到 |
|---|---|---|
| 极品 | 100%（满效果） | 笔画数、字形、落笔节奏都到位 |
| 良品 | 80% | 基本到位、略有瑕疵或偏慢 |
| 劣质 | 55% | 明显连笔/拖沓 |
| 废符 | 0%（不施放、消耗符） | 写得太差或认不准 |

- **评分（客观）**：识别置信度（立方加权，越低扣得越狠）、简化笔画数、点数、书写时长，与每个字的"标准范本"比对得出综合分；阈值 ≥95% / ≥75% / ≥50% 分为四档。
- **缩放方式**：火/雷/护 按倍率缩放伤害/护盾；净 的品质体现在"清除负面条数（3/2/1）+ 圣化伤害"。
- **品质写在黄符物品上**：tooltip 直接显示「品质: 极品/良品/劣质/废符」（无需 dev、无需使用即可见）；**同组合 + 同类型 + 同品质的符可叠加**（每组 64）。
- **结构废符（乱写没成式）与品质废符（成式但分太低）统一**：右键都消耗并 action bar 提示「废符」。
- **实现分工**：input 负责客观评分并定品质、写进物品；sigillum 负责把品质换成效果倍率施放（倍率/阈值是各自一处常量，易调）。

### 修饰字规则

修饰字本身不生效，全局作用于黄符上**所有技能字**：

| 字 | 修饰效果 |
|---|---|
| 强 | 所有技能字的伤害/控制等级/护盾量 +50%，冷却 +30% |
| 续 | 所有技能字的持续时间 +60% |
| 疾 | 所有技能字的前摇 -40% |
| 广 | 所有技能字从单体变范围（半径 4 格），范围效果降为 70% |

### 刻印符全量组合

刻印符第一格固定为「刻」，以目标方块为中心持续生效。

#### 刻印叠加机制（全技能通用）

所有刻印都支持**后续强化符叠加**。方块上已有刻印后，再拿同技能字的刻印符右键该方块不覆盖，而是累加强化。**所有叠加都有硬上限或递减收益，防止无限堆数值**：

- **续时叠加**：持 `刻 + X + 续` 右键已有 X 刻印的方块 → 剩余时间 **+5 分钟**，但单个刻印**总时长封顶 30 分钟**；到顶后续符只把剩余时间刷回上限，不再增加。
- **范围叠加**：持 `刻 + X + 广` 右键已有 X 刻印的方块 → 半径 **+1 格**，最多叠加 **5 次**（半径上限 3+5=8 格）。
- **强度叠加**：持 `刻 + X + 强` 右键 → 效果强度递减增益（**第 1 次 +50%、第 2 次 +25%、第 3 次 +10%**），强度增益**上限 +85%**，之后不再提升。
- **速度叠加**：持 `刻 + X + 疾` 右键 → 脉冲周期缩短一档，但有**周期下限**（最快 0.5 秒/脉冲），到下限不再缩短。

**叠加限制**：
- 任一刻印的**总叠加次数上限 8 次**，达到上限后所有强化符只刷新续时（受 30 分钟封顶约束），不再产生新增益。
- 只能叠加同技能字的符（`刻 + 镇` 刻印上不能再放 `刻 + 火`，提示"此处已有刻印：镇"并拒绝）。
- 首次创建时若三格含修饰字，修饰字既参与初始参数又算一次叠加计数。
- 刻印时间耗尽消失后所有叠加清零，必须从头建。

#### 刻 + 修饰字 + 技能字（三格）

等同于 `刻 + 技能 + 修饰`，三格无序。

#### 刻 + 修饰字 + 修饰字（三格）

第二、三格都是修饰字时，符上没有任何技能字可供刻印/修饰 → **废符**（与"修饰字单独成符为废符"同理）。刻印符的格式天然只容纳一个修饰字（`刻 + 技能字 + 修饰字`），不存在多修饰字争抢优先级的情况。

#### 防护罩特殊规则（刻 + 护 专用）

`刻 + 护` 的防护罩区别于其他刻印：

- **每次击退消耗时间**：每击退一只亡灵消耗 20 秒。其他刻印不受攻击影响，只等时间自然耗尽。
- **失效后重置**：时间耗尽后若想再建必须从头开始，之前的范围/强度叠加全部清零。

叠加机制与所有刻印通用。

#### 刻 + 数字字（二格）

数字字不是技能字、不是修饰字，刻印符后两格含数字字 → 废符。

### 指定符

指定符只用于快速吟唱绑定，不产生战斗效果：

| 组合 | 结果 |
|---|---|
| 数字 + 技能字 + 空 | 将技能字绑定到该数字 |
| 数字 + 空 + 空 | 解除该数字的绑定 |

### 废符

以下情况无法形成有效黄符，右键产生轻微火花：
- 三格全空
- 普通符没有任何技能字（纯修饰字 / 纯数字字）
- 同一玩法字在一张符中重复出现（重复技能字或重复修饰字）
- 刻印符后两格含数字字
- 第一格为数字字但格式不匹配指定符（如数字 + 数字 + 空）
- 刻印符后两格没有任何技能字（如 `刻 + 强 + 空` 或只有第一格「刻」）

## 六、评分与黄符品级

手写质量决定黄符品级。`aozaink-core` 已输出：

- `confidence`（置信度）
- `simplifiedStrokeCount`（简化后笔画数）
- `simplifiedPointCount`（简化后点数）
- `writingDurationMs`（书写耗时）

sigillum 接收 `InkRecognizedEvent` 后，把每个字的识别质量打包成**符咒评分**。评分规则初步设计：

1. **置信度满分**：模型对标准打印体的 confidence 作为 100% 基准；
2. **笔画完整度**：玩家手写笔画数与标准打印体笔画数的接近程度；
3. **书写节奏**：过快或过慢都扣分，标准打印体书写耗时作为参考基准；
4. **点数合理度**：点过少像涂鸦，点过多像乱画，与标准打印体比较。

最终评分把每个字的得分平均，得到整张黄符的**品级分**：

| 品级 | 分数区间 | 效果 |
|---|---|---|
| **下品符** | 0% – 49% | 效果只发挥 60%，持续时间缩短 30%，刻印持续时间减半 |
| **中品符** | 50% – 79% | 正常效果 |
| **上品符** | 80% – 100% | 效果提升 20%，持续时间延长 20%，刻印时范围略微扩大 |

> 标准打印体数据需要先在游戏外采集：用标准字体生成字帖，喂给 core 的识别器，记录 confidence、笔画数、点数、耗时，作为 100% 基准。玩家手写再与这个基准比较。

> 实现落点：黄符 `CustomData` 需要额外存储每格的 `confidence`、`simplifiedStrokeCount`、`simplifiedPointCount`、`writingDurationMs`，以及总评分和品级。当前 input 只保存了三格字与符类型，sigillum 拿不到评分依据，需要后续实现方案确认。

## 七、黄符两种使用方式

### 1. 直接施法

主手持已成符黄符，右键释放：

- 对实体释放时，术式直接作用于该目标（单体）；
- 对空气/地面释放时，术式以玩家前方为中心生效；
- 作用范围由符上技能字效果 + 修饰字共同决定：
  - 默认单体；
  - 含「广」时扩展为范围或影响多个目标；
  - 含「疾」时出手更快；
- 对方块释放时，若黄符是刻印符则进入刻印流程，否则按"对地面释放"处理。

### 2. 刻印到方块

持刻印符（「刻」开头的符）潜行右键方块，把术式刻印在方块上：

- 一个方块**只能同时存在一个刻印**；
- 若目标方块已有同技能刻印，则按叠加机制累加强化而不覆盖；
- 若目标方块已有不同技能刻印，提示"此处已有刻印：X"并拒绝；
- 刻印后黄符被消耗；
- 方块被破坏或长期卸载时，刻印失效。

刻印符第一格固定为「刻」。后两格无序解析：
- **一个技能字**：产生对应技能的周期性脉冲效果；
- **一个技能字 + 一个修饰字**：修饰字全局修饰该刻印的技能效果；
- **两个不同技能字**：增加第二个独立脉冲；
- **没有技能字 / 重复玩法字**：废符。

> 实现落点：刻印状态需要明确存储方案，例如按维度用 `LevelSavedData` 以 `BlockPos` 为键保存 `SealAnchorState`，或给刻印方块附加组件。当前设计稿只定义玩法规则，具体存储模型需要后续实现方案确认。

## 八、快速吟唱字

玩家可以把一个常用字绑定到中文数字字 一～九，例如：

- `/sigillum dev` 开启开发模式后，用 `/sigillum bind 1 雷` 把"雷"绑定到"一";
- 战斗中写一个"一"字，自动视为"雷"，缩短前摇。

快速吟唱规则：

- 写绑定字时，效果等同写原字;
- 前摇缩短 30%，但法耗或冷却略微增加（若后续加入资源系统）;
- 若当前未绑定，则按字面意思识别。

## 九、不做清单

首版明确不做以下内容，避免工程爆炸：

- 不做召唤生物/宠物；
- 不做大型多方块仪式；
- 不做跨维度远程维持；
- 不做几十个字的大字库；
- 不做法力/灵力资源系统（以冷却为主，资源系统后续接入）；
- 不做玩家之间的符咒交易/经济系统；
- 不做复杂的 NPC/尸系 AI。

## 十、体验节奏

一次完整的符术循环应该是：

1. 玩家准备空白黄符（3 纸 + 1 黄色染料合成 6 个）；
2. 把黄符放在工作台上，右键打开三格书写 UI；
3. 用鼠标在三格里手写字，每格停止书写 0.7 秒后自动识别；
4. 若要做刻印符，第一格必须写「刻」；
5. 点击"成符"，黄符被写入三格字与品级分；
6. 主手使用黄符施法，或持刻印符潜行右键方块刻印；
7. 观察效果，下次调整写法以提高品级。

失败也应有反馈：

- 识别失败：黄符变成"废符"，右键只产生轻微火花；
- 品级太低：效果打折，玩家能从 HUD 看到自己写的是"下品"；
- 多修饰字：普通符上多个不同修饰字全局叠加生效（受三格限制最多 2 个，强/续/疾/广分属不同维度，互不冲突）；同一修饰字重复写判废符。刻印符格式只容纳一个修饰字，写两个修饰字即废符。

## 十一、后续扩展方向

首版骨架跑通后，可以逐步加入：

- 更多技能字：缚、破、冰、风、毒、愈等；
- 更多修饰字：几何形态字点/线/场/贴；
- 刻印进阶：多个刻印相邻时形成复合阵法；
- 资源系统：灵力/法力消耗，与 core 的 `powerMultiplier` 挂钩；
- 敌人抗性：Boss 对控制效果有抗性，亡灵对镇/净更脆弱；
- 数据包支持：字定义、联结规则、FX 预设可通过 JSON 热更新。

## 十二、设计原则

- **服务端权威**：所有效果结算在服务端执行，客户端只负责输入与表现；
- 识别结果可来自客户端（core 在客户端完成 OCR），但效果结算与刻印状态必须服务端权威；
- **数据驱动**：字表与联结表尽量用配置文件或数据包描述，减少硬编码；
- **线性可预测**：默认情况下三格效果线性叠加；
- **反馈清晰**：玩家能知道自己写的是什么品级、会触发什么术式、为什么失败；
- **扩展友好**：新增字或联结不应改动现有编译器骨架。
